-------------------

The size word in the header reflects the body length:
  PLAY3  : 0x0510? header field corresponds to body 0x550

Current best interpretation
---------------------------
- Best size match among currently available files: step04 (+6 bytes)
- Best overlap mismatch count among currently available files: step07 (1059 mismatches)

So there are currently two "best" candidates depending on metric:
- step04 is best for total body length
- step07 is best for byte agreement within the overlapping range

Important note
--------------
Earlier work reached a size-matched state (0x550 body) in a transient step05,
but those step05 files are not currently present in this environment.
Therefore, the present report is limited to the files that are available now.

Conclusion
----------
Algorithm recovery is effectively complete, but binary-identical reconstruction
is still blocked by exact encoding details such as:

- short vs long branch selection
- XASM addressing forms like (!label), [label], [(!label)]
- operand encoding in self-modifying sections
- front-half parser / scheduler instruction forms

Recommended next reconstruction target:
  use step04/step07 as bases and continue instruction-by-instruction
  normalization around entry, mml_conv, init/init2, and self-modifying mixer setup.
