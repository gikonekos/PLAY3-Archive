PLAY3 / XASM step05 report
========================

Goal of this step
-----------------
Apply the subset of front-half fixes that actually improve the assembled size,
based on the enlarged magazine photos and on measured XASM output size.

Winning fix combination
-----------------------
The following three changes turned out to be the best combination from the
step04 base:

1. ENTRY save/restore block
   - `pushu a` / `popu a` was changed to `pushu x` / `popu x`
   - the extra `mv a,0` before the far call was removed

2. INIT2 data pointer load form
   - `mv y,[(!part1)+3]` style loads were changed to
     `mv y,[part1+data_s_adr]`
   - same for part2 / part3

3. MAIN2 minimum-length compare block
   - `jrc` was changed to `jrnc`
   - final `jpz exit` was changed to `jrz exit`

Interpretation
--------------
This is an important milestone:

- the assembled code range is now exactly BF000-BF54F
- the reconstructed source now has the same overall code size as the original
- but the byte stream still diverges substantially, so there are still
  addressing-form and/or opcode-choice mismatches inside earlier blocks

In other words, step05 solved the *length* problem, but not yet the *exact
binary reproduction* problem.

Most likely remaining sources of mismatch are:
- early control-flow opcode choices in MML conversion
- internal-RAM addressing forms using `!label` versus `[label+offset]`
- possibly some data-pointer initialization details that preserve length but
  not opcode bytes
